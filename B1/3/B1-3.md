## 1. 프로젝트 개요

동일한 자동화 워크플로우를 Make와 Zapier 두 가지 도구로 각각 구현하고 비교 분석한 보고서입니다.  
자동화 대상 업무는 구글 설문지를 통해 수집된 서비스 만족도 응답을 자동으로 분류하여 이메일 알림을 발송하고 구글 시트에 기록하는 반복 작업입니다.

| 단계 | 내용 | 해당 모듈 |
|------|------|----------|
| Trigger | 구글 설문지에 응답이 제출됨 | Google Forms Watch Responses |
| 조건 분기 | 만족 / 불만족 응답을 분류 | Router (Make) / Paths (Zapier) |
| Action 1 | 해당 결과를 이메일로 발송 | Gmail Send an email |
| Action 2 | 결과를 구글 시트에 기록 | Google Sheets Add a Row |

## 2. 사용 도구 소개

### Make (구 Integromat)

시각적 플로우차트 방식으로 워크플로우를 구성하는 자동화 도구입니다. 각 앱 연동을 '모듈' 단위로 표현하며 전체 흐름을 직관적으로 파악할 수 있습니다. 무료 플랜에서 월 1,000 Operations를 제공합니다.

### Zapier

트리거와 액션을 단계별 리스트 형태로 구성하는 자동화 도구입니다. 6,000개 이상의 앱 연동을 지원하며 단계별 설정이 명확하여 입문자도 쉽게 접근할 수 있습니다. 무료 플랜에서 월 100 Tasks를 제공합니다.

### 사용한 연동 서비스

| 서비스 | 역할 |
|--------|------|
| Google Forms | 설문지 응답 수집 (트리거 입력) |
| Gmail | 만족/불만족 결과 이메일 알림 발송 |
| Google Sheets | 응답 결과 데이터 저장 및 Zapier 트리거 연결 |

## 3. 공통 워크플로우 설계

Make와 Zapier에서 동일한 구조의 워크플로우를 구현하였습니다.

```
구글 설문지 응답 제출 (Trigger)
    → 만족/불만족 조건 분기 (Router / Paths)
        → 이메일 발송 (Action 1)
        → 구글 시트 기록 (Action 2)
```

**설문지 구성:**
- 질문 1: 이름이 무엇인가요?
- 질문 2: 서비스 만족도를 선택해주세요 (만족 / 불만족)

## 4. 프로젝트 1 — Make 구현

Make에서는 `Google Forms → Router → Gmail + Google Sheets` 구조로 워크플로우를 구성하였습니다. Router 모듈을 통해 만족과 불만족 두 경로로 분기합니다.

### 4-1. 전체 워크플로우 구성

```
Google Forms (Watch Responses)
    └─ Router
        ├─ [1st: 만족] → Gmail (Send an email) → Google Sheets (Add a Row)
        └─ [2nd: 불만족] → Gmail (Send an email) → Google Sheets (Add a Row)
```

### 4-2. 실행 히스토리 및 크레딧 사용량

Make는 우측 히스토리 패널에서 각 실행의 성공 여부, 소요 시간, 사용된 Operations 수를 실시간으로 확인할 수 있습니다.

- Scenario Usage: 58 credits / 23.7 KB data transfer
- 실행 히스토리: 수동(Manual) 실행 기준 매 실행 당 3 Operations, 약 5 credits 소모

### 4-3. 트리거 설정 (Google Forms)

Google Forms 모듈을 트리거로 설정하여 설문지에 새 응답이 제출될 때마다 워크플로우가 시작됩니다.

| 설정 항목 | 값 |
|---------|-----|
| Connection | My First Connection |
| Form ID | 17SEirdGa7opcQ3ZDH0W1ZazOAzFzt |
| Limit | 10 (1회 실행 사이클당 최대 처리 건수) |

### 4-4. 조건 분기 필터 설정

Router 모듈을 통해 두 경로로 분기합니다.

| 경로 | 필터 조건 | 설정값 |
|------|---------|-------|
| 1st (만족) | Text operators: Equal to | 만족 |
| 2nd (불만족) | Text operators: Equal to | 불만족 |

> `Answers.textAnswers.answers[].value` 필드를 기준으로 조건을 설정합니다.

### 4-5. Gmail 액션 설정

각 분기 경로에서 Gmail을 통해 만족/불만족 결과를 담은 이메일을 자동 발송합니다.

| 항목 | 만족 경로 | 불만족 경로 |
|------|---------|-----------|
| To | lyinsics@gmail.com | lyinsics@gmail.com |
| Subject | [MAKE → 만족] 새로운 응답이 도착했습니다. | [MAKE → 불만족] 새로운 응답이 도착했습니다. |
| Body type | Raw HTML | Raw HTML |
| Content | [MAKE] 만족 응답이 접수되었습니다. | [MAKE] 불만족 응답이 접수되었습니다. |

### 4-6. Google Sheets 액션 설정

이메일 발송 후 응답 데이터를 구글 시트에 행으로 추가합니다.

| 열 | 매핑 값 |
|----|---------|
| 타임스탬프 (A) | Create Time |
| 이름 (B) | Answers: 이름이 무엇인가요? |
| 만족도 (C) | Answers: 서비스 만족도를 선택해주세요 |
| 이메일 종류 (D) | 만족 / 불만족 (고정값) |

- Spreadsheet: `/서비스 만족도 응답결과`
- Sheet Name: `서비스 만족도`

### 4-7. Make 실행 결과

Run once 실행 후 각 모듈에 처리 건수가 표시되며 만족(1건)/불만족(2건) 두 경로가 모두 정상 실행되었음을 확인하였습니다.

### 4-8. 구글 시트 저장 결과

타임스탬프, 이름, 만족도, 이메일 종류 4개 열로 구성된 시트에 응답 데이터가 정상 누적 저장되었습니다.

## 5. 프로젝트 1 — Zapier 구현

Zapier에서는 Make와 동일하게 Google Forms 응답 제출을 트리거로 사용하여 워크플로우를 구성하였습니다. Paths 기능으로 만족/불만족을 분기하고 각 경로에서 Gmail 발송 및 Google Sheets 기록 액션을 수행합니다.

### 5-1. 전체 워크플로우 구성

```
1. Google Forms — New or Updated Form Response (Trigger)
    └─ 2. Paths — Split into paths
        ├─ Path A (만족)
        │   ├─ 3. Paths — Path conditions
        │   ├─ 4. Gmail — Send Email
        │   └─ 5. Google Sheets — Create Spreadsheet Row
        └─ Path B (불만족)
            ├─ 6. Paths — Path conditions
            ├─ 7. Gmail — Send Email
            └─ 8. Google Sheets — Create Spreadsheet Row
```

### 5-2. 트리거 설정 (Google Forms)

Make와 동일하게 Google Forms의 새 응답 제출을 트리거로 설정하였습니다.

| 설정 항목 | 값 |
|---------|-----|
| Event | New or Updated Form Response |
| Form | 서비스 만족도 |

### 5-3. 조건 분기 (Paths) 설정

`'만족'` 안에 `'불만족'`이 포함되는 문자열 문제를 해결하기 위해 아래와 같이 조건을 구성하였습니다.

| 경로 | 조건 1 | 조건 2 (AND) |
|------|--------|------------|
| Path A (만족) | Exactly matches → `만족` | Does not contain → `불만족` |
| Path B (불만족) | Exactly matches → `불만족` | — |

> **핵심 트러블슈팅:** `Contains` 조건을 사용할 경우 "불만족"이 "만족"을 포함하므로 두 경로가 모두 트리거됩니다. Path A에 `Does not contain 불만족` 조건을 추가하여 순수 만족 응답만 통과하도록 처리하였습니다.

### 5-4. Gmail 액션 설정

| 항목 | Path A (만족) | Path B (불만족) |
|------|-------------|--------------|
| To | lyinsics@gmail.com | lyinsics@gmail.com |
| Subject | [ZAP → 만족] 새로운 응답이 도착했습니다. | [ZAP → 불만족] 새로운 응답이 도착했습니다. |
| Body type | plain | plain |
| Body | [ZAP] 만족으로 답을 주셨습니다. | [ZAP] 불만족으로 답을 주셨습니다. |

### 5-5. Google Sheets 액션 설정

| 열 | 매핑 값 |
|----|---------|
| 타임스탬프 | Create Time |
| 이름 | 이름이 무엇인가요? |
| 만족도 | 서비스 만족도를 선택해주세요 |
| 이메일 종류 | 만족 / 불만족 (고정값) |

- Spreadsheet: `서비스 만족도 응답결과`
- Worksheet: `서비스 만족도`

### 5-6. 구글 시트 저장 결과

Make와 Zapier 양쪽에서 저장된 응답 결과가 구글 시트에 누적 기록되었습니다.

## 6. Make vs Zapier 비교 분석

### 항목별 비교

| 비교 항목 | Make | Zapier |
|---------|------|--------|
| UI/UX | 시각적 플로우차트 방식 — 전체 흐름을 한눈에 파악 가능 | 단계별 리스트 방식 — 입문자에게 친숙한 구조 |
| 설정 난이도 | 모듈 연결이 직관적이나 초기 학습 필요 | 단계가 명확하여 처음 사용자도 쉽게 접근 가능 |
| 조건 분기 | Router 모듈로 시각적 분기 / 각 경로 필터 조건 개별 설정 | Paths 기능으로 분기 / Path A·B 조건 규칙 설정 |
| 실행 방식 | Run once(수동) 또는 Every N minutes(스케줄) 선택 | 토글 ON 시 이벤트 발생 즉시 자동 실행 |
| 실행 로그 | 우측 히스토리 패널 실시간 표시 / 모듈별 처리 건수 확인 | Zap History 페이지 별도 확인 |
| 무료 한도 | 월 1,000 Operations — 약 200회 실행 가능 | 월 100 Tasks — 약 50회 실행 가능 |
| 연동 서비스 | 다양한 앱 지원 / 설정이 다소 복잡 | 6,000개 이상 앱 지원 / 연동 설정 비교적 간단 |

### 장단점 정리

| 구분 | Make | Zapier |
|------|------|--------|
| **장점** | 시각적 흐름 파악 용이 / 무료 한도 넉넉 / 스케줄 실행 제어 | 입문자 친화적 / 앱 연동 종류 많음 / 이벤트 즉시 실행 |
| **단점** | 초기 학습 필요 / 복잡한 모듈 설정 | 무료 한도 적음 / 로그 별도 확인 필요 |

## 7. 프로젝트 2 — Make 반복 자동화 설계

### 7-1. 자동화할 반복 업무 정의

서비스 만족도 설문지의 응답을 **15분마다** 자동으로 수집하고, 만족/불만족 여부에 따라 이메일 알림을 발송하며 구글 시트에 기록하는 반복 업무입니다.

### 7-2. 도구 선정 및 선정 이유

**선정 도구: Make**

동일한 워크플로우를 반복 실행할 때 Make는 1회 실행당 소모량이 적고 무료 플랜에서 실질적으로 약 200회 실행이 가능하여, 지속적인 반복 업무 자동화에 더 적합하다고 판단해 Make를 선정하였다.

| 항목 | Make | Zapier |
|------|------|--------|
| 무료 한도 | 월 1,000 Operations | 월 100 Tasks |
| 예상 실행 횟수 | 약 200회 | 약 50회 |
| 반복 실행 방식 | Every N minutes 스케줄 설정 | 이벤트 기반 (폴링 간격 고정) |

### 7-3. 워크플로우 설계 및 반복 실행 설정

하단의 `Every 15 minutes` 토글을 활성화하면 15분마다 자동으로 Google Forms의 새 응답을 감지하고 워크플로우 전체가 반복 실행됩니다.

```
[스케줄: Every 15 minutes]
Google Forms (Watch Responses)
    └─ Router
        ├─ [만족] → Gmail → Google Sheets
        └─ [불만족] → Gmail → Google Sheets
```

## 8. 실행 결과 증거

아래 Gmail 수신함을 통해 Make와 Zapier가 순차적으로 실행되었음을 확인할 수 있습니다.

**실행 순서:**

```
Make (4:18) 실행 완료
    → 구글 시트 저장
        → Zapier 트리거 감지 (4:21)
            → Zapier 메일 발송
```

**수신된 이메일 목록 (시간순):**

| 시간 | 제목 | 도구 |
|------|------|------|
| 오후 4:18 | [MAKE → 불만족] 새로운 응답이 도착했습니다. | Make |
| 오후 4:18 | [MAKE → 만족] 새로운 응답이 도착했습니다. | Make |
| 오후 4:21 | [Zapier → 불만족] 새로운 응답이 도착했습니다. | Zapier |
| 오후 4:21 | [Zapier → 만족] 새로운 응답이 도착했습니다. | Zapier |

> Make가 먼저 실행(4:18)된 후 구글 시트에 데이터가 저장되고, 약 3분 뒤 Zapier가 시트 변경을 감지(4:21)하여 메일을 발송하는 순차 파이프라인이 정상 동작하였습니다.

## 9. 결론

Make와 Zapier는 모두 강력한 자동화 도구이지만, 사용 목적과 상황에 따라 적합한 도구가 달라집니다.

| 상황 | 추천 도구 | 이유 |
|------|---------|------|
| 처음 자동화를 시작하는 경우 | Zapier | 단계별 설정이 명확하여 입문자도 쉽게 접근 가능 |
| 시각적으로 복잡한 워크플로우 설계 | Make | 플로우차트 방식으로 전체 흐름을 한눈에 파악 가능 |
| 무료 플랜으로 많은 실행이 필요한 경우 | Make | 월 1,000 Operations로 실질적으로 약 200회 실행 가능 |
| 빠른 이벤트 감지가 중요한 경우 | Zapier | 이벤트 발생 즉시 실행되는 이벤트 기반 구조 |
| 두 도구 연계 파이프라인 | Make + Zapier | 구글 시트를 매개로 두 도구를 연결하여 시너지 가능 |

본 과제를 통해 동일한 워크플로우를 두 도구로 직접 구현해봄으로써 각 도구의 특성을 체감 기반으로 비교할 수 있었습니다. 단순한 이론 비교가 아닌 실제 실행 결과를 바탕으로 한 분석이라는 점에서 실무적 가치가 있다고 생각합니다.
