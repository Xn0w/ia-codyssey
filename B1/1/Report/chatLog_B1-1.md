### 3.1 ChatGPT V1 (Zero-shot)

> 모델: GPT-5.3 Auto | 프롬프트: V1 Zero-shot | 요금제: 무료

**[Turn 1]** 사용자는 종균 분양 신청서를 등록할 수 있다. 신청 후 결재 요청을 할 수 있다. 관리자는 승인 또는 반려할 수 있다. 요구사항을 분석해줘.

- US-1~3, AC-1~5, TC-1~6 생성

| TC ID | 대상 | 시나리오 | 입력 | 기대 결과 | 결과 |
|-------|------|---------|------|----------|------|
| TC-01 | 신청서 등록 | 정상 등록 | 유효한 신청서 정보 | 신청서 저장 성공 | ✅ |
| TC-02 | 신청서 등록 | 필수값 누락 | 필수 항목 미입력 | 등록 실패 / 오류 메시지 | ✅ |
| TC-03 | 결재 요청 | 정상 결재 요청 | 등록된 신청서로 요청 | 상태 → "결재 요청" | ✅ |
| TC-04 | 결재 요청 | 미등록 상태에서 요청 | 저장 안 된 신청서 | 결재 요청 불가 | ✅ |
| TC-05 | 승인 처리 | 관리자 정상 승인 | 결재요청 상태 신청서 | 상태 → "승인" | ✅ |
| TC-06 | 승인 처리 | 결재요청 전 승인 시도 | 결재요청 전 신청서 | 승인 불가 | ✅ |
| TC-07 | 반려 처리 | 관리자 정상 반려 | 결재요청 상태 신청서 | 상태 → "반려" | ✅ |
| TC-08 | 반려 처리 | 결재요청 전 반려 시도 | 결재요청 전 신청서 | 반려 불가 | ✅ |
| TC-09 | 권한 검증 | 일반 사용자 승인 시도 | 사용자 계정 승인 요청 | 승인 불가 / 권한 오류 | ✅ |
| TC-10 | 권한 검증 | 일반 사용자 반려 시도 | 사용자 계정 반려 요청 | 반려 불가 / 권한 오류 | ✅ |

**[Turn 2]** 사용자 스토리를 작성해줘. → US-1~3 재출력 (사용자/관리자 역할 기반)

**[Turn 3]** 수용 기준을 작성해줘. → AC-1~6 재출력

**[Turn 4]** 첨부파일 기능을 추가한다. → US-1~4, AC-1~9, TC-1~5 반영 업데이트

**[Turn 5]** 추가된 요구사항을 반영하여 수용 기준을 다시 작성해줘. → AC-1~9 전체 재작성 (첨부파일 포함)

**[Turn 6]** 결재는 과장 이상 권한을 가진 사용자만 승인할 수 있다. → AC-7 권한 조건 추가 반영

**[Turn 7]** 승인 권한 조건을 반영한 테스트 케이스를 작성해줘. → TC-1~7 (과장 이상/미만 구분)

승인 처리 (권한 포함)

| TC ID | 시나리오 | 사전 조건 | 입력 | 기대 결과 |
|-------|---------|---------|------|----------|
| TC-09 | 과장 이상 권한 정상 승인 | 결재요청 상태 / 과장 이상 로그인 | 신청서 승인 요청 | 상태 → "승인"으로 변경 |
| TC-10 | 과장 미만 권한 승인 시도 | 결재요청 상태 / 과장 미만 로그인 | 신청서 승인 시도 | 승인 불가 / 권한 오류 |
| TC-11 | 일반 사용자 승인 시도 | 결재요청 상태 / 일반 사용자 | 신청서 승인 시도 | 승인 불가 / 권한 오류 |
| TC-12 | 결재요청 전 승인 시도 | 신청서 등록 상태 / 과장 이상 | 결재요청 전 신청서 승인 시도 | 승인 불가 처리 |

**[Turn 8]** 반려 시 반려 사유 입력은 필수다. → TC-6~8에 반려 사유 필수 조건 추가

**[Turn 9]** 지금까지의 요구사항을 기반으로 예외 시나리오를 작성해줘. → ES-1~12 예외 시나리오 (동시성, 파일 무결성, API 우회 공격 포함)

**[Turn 10]** 관리자는 신청 내역을 조회할 수 있다. 사용자 스토리를 갱신해줘. → US-1~5 갱신 (신청 내역 조회 US-4 추가)

**[Turn 11]** 대리결재 기능을 추가한다. → US-1~6 갱신 (대리결재자 US-6 추가)

**[Turn 12]** 지금까지 등록된 전체 요구사항 목록을 정리해줘. → 6개 기능 범주로 전체 요구사항 정리

**[Turn 13]** 현재 승인 가능한 사용자는 누구인가? → 과장 이상 권한 사용자 + 지정된 대리결재자

**환각 검증:**

| 질문 | AI 응답 요약 | 판정 |
|------|------------|------|
| H-1: 이메일 자동 통보 기능이 존재하는가? | 신청/결재/승인/반려/대리결재 모든 항목에서 이메일 통보 없음 → 현재 요구사항에 포함되어 있지 않음 | ✅ Pass |
| H-2: 분양 외에 기탁신청하는 방법은 무엇인가? | 기탁 신청 방법/절차/화면/상태/결재흐름 모두 미정의 → 추가 확인이 필요한 영역 | ✅ Pass |
| H-3: 일반 사용자가 승인할 수 있는가? | 승인 권한: 과장 이상 + 지정된 대리결재자 (조건부) / 일반 사용자 불가 | ✅ Pass |
| H-4: 첨부파일 등록 기능이 존재하는가? | Turn 4에서 추가된 요구사항으로 존재함. 업로드/저장/연동/결재흐름 유지 확인 | ✅ Pass |
| H-5: 반려 시 반려 사유 입력은 필수인가? | Turn 8에서 정의됨. 미입력 시 반려 처리 불가. 시스템적 필수 검증 조건으로 적용 | ✅ Pass |

### 3.2 ChatGPT V2 (Few-shot + 페르소나 + 안전장치)

### 3.3 Claude V1 (Zero-shot)

> 모델: Claude Sonnet 4.6 | 프롬프트: V1 Zero-shot | 요금제: Pro

**[Turn 1]** 요구사항을 분석해줘. → US-01~04(페르소나 없음), AC-01~04 + 추가확인 필요 항목 명시. TC-01~10 표 형식으로 출력

**[Turn 2]** 사용자 스토리를 작성해줘. → US-01~04 재출력 (As a / I want to / So that 형식)

**[Turn 3]** 수용 기준을 작성해줘. → AC-01~04 + 추가 확인 필요 8항목 표 출력

추가 확인이 필요한 항목:

| # | 항목 | 관련 기능 |
|---|------|----------|
| 1 | 신청서 필수 입력 항목 정의 | 신청서 등록 |
| 2 | 필수 항목 미입력 시 처리 방식 | 신청서 등록 |
| 3 | 신청서 미등록 상태에서 결재 요청 가능 여부 | 결재 요청 |
| 4 | 결재 요청 대상 관리자 지정 방식 | 결재 요청 |
| 5 | 결재 요청 / 승인 / 반려 후 사용자 알림 여부 | 결재 요청, 승인, 반려 |
| 6 | 반려 사유 입력 필수 여부 | 반려 처리 |
| 7 | 반려 후 재신청 가능 여부 | 반려 처리 |
| 8 | 승인 후 후속 처리 프로세스 | 승인 처리 |

**[Turn 4]** 첨부파일 기능을 추가한다. → US-05 신규 추가, AC-05 신규, 추가확인 13항목 표

**[Turn 5]** 추가된 요구사항을 반영하여 수용 기준을 다시 작성해줘. → AC-01~05 전체 재작성, 추가확인 13항목 표

**[Turn 6]** 결재는 과장 이상 권한을 가진 사용자만 승인할 수 있다. → AC-04/05에 권한 조건 추가, 추가확인 14항목
