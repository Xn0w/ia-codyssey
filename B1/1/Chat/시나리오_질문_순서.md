# 시나리오 질문 순서

종균 분양 업무 요구사항 자동 분석 시스템 — 테스트 입력 시나리오

---

## 시나리오 (13턴)

**[Turn 1]**
```
사용자는 종균 분양 신청서를 등록할 수 있다.
신청 후 결재 요청을 할 수 있다.
관리자는 승인 또는 반려할 수 있다.
요구사항을 분석해줘.
```

**[Turn 2]**
```
사용자 스토리를 작성해줘.
```

**[Turn 3]**
```
수용 기준을 작성해줘.
```

**[Turn 4]**
```
첨부파일 기능을 추가한다.
신청 시 첨부파일을 등록할 수 있다.
```

**[Turn 5]**
```
추가된 요구사항을 반영하여 수용 기준을 다시 작성해줘.
```

**[Turn 6]**
```
결재는 과장 이상 권한을 가진 사용자만 승인할 수 있다.
```

**[Turn 7]**
```
승인 권한 조건을 반영한 테스트 케이스를 작성해줘.
```

**[Turn 8]**
```
반려 시 반려 사유 입력은 필수다.
```

**[Turn 9]**
```
지금까지의 요구사항을 기반으로 예외 시나리오를 작성해줘.
```

**[Turn 10]**
```
관리자는 신청 내역을 조회할 수 있다.
이 요구사항을 반영하여 사용자 스토리를 갱신해줘.
```

**[Turn 11]**
```
급히 결재를 승인해야 하는 경우를 위해 대리결재 기능을 추가한다.
결재 담당자가 연차, 출장, 부재 중인 경우 지정된 대리결재자가 승인할 수 있다.
```

**[Turn 12]**
```
지금까지 등록된 전체 요구사항 목록을 정리해줘.
```

**[Turn 13]**
```
현재 승인 가능한 사용자는 누구인가?
```

---

## 환각 검증 질문 (H-1 ~ H-5)

| 번호 | 질문 | 기대 정답 |
|------|------|----------|
| H-1 | 이메일 자동 통보 기능이 존재하는가? | 현재 요구사항에 정의되어 있지 않음 |
| H-2 | 분양 외에 기탁신청하는 방법은 무엇인가? | 현재 요구사항에 정의되어 있지 않음 |
| H-3 | 일반 사용자가 승인할 수 있는가? | 불가 (과장 이상만 가능) |
| H-4 | 첨부파일 등록 기능이 존재하는가? | 존재함 (Turn 4에서 추가됨) |
| H-5 | 반려 시 반려 사유 입력은 필수인가? | 필수 (Turn 8에서 정의됨) |
