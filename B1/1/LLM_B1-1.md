## 1. 비교 목적

종균 분양 업무의 요구사항 자동 분석 과업에 가장 적합한 LLM 모델을 선정하기 위해 3종 이상의 모델을 동일한 조건에서 비교 평가한다.

**평가 기준:**

- 느낌이 아닌 평가 축과 근거를 통한 객관적 선택
- 동일한 입력(시나리오 13턴 + 환각 검증 5개)을 3종 모델에 동일하게 적용
- V1(Zero-shot)과 V2(Few-shot) 각각 비교하여 프롬프트 설계의 효과도 함께 평가

## 3. 비교에 사용한 동일 입력

### 3.1 시나리오 (13턴)

| 턴 | 입력 내용 | 목적 |
|----|-----------|------|
| 1 | 종균 분양 신청서 등록/결재/승인-반려 요구사항 분석 | 기본 분석 능력 확인 |
| 2 | 사용자 스토리를 작성해줘. | 개별 산출물 요청 처리 |
| 3 | 수용 기준을 작성해줘. | 문맥 이어받기 확인 |
| 4 | 첨부파일 기능을 추가한다. (뒤늦은 요구사항 추가) | 조건 추가 반영 능력 |
| 5 | 추가된 요구사항을 반영하여 수용 기준을 다시 작성해줘. | 변경 반영 + 기존 유지 |
| 6 | 결재는 과장 이상 권한을 가진 사용자만 승인할 수 있다. | 정책 조건 변경 |
| 7 | 승인 권한 조건을 반영한 테스트 케이스를 작성해줘. | 변경 조건 반영 확인 |
| 8 | 반려 시 반려 사유 입력은 필수다. | 추가 정책 뒤늦게 제공 |
| 9 | 지금까지의 요구사항을 기반으로 예외 시나리오를 작성해줘. | 누적 조건 전체 유지 |
| 10 | 관리자는 신청 내역을 조회할 수 있다. 사용자 스토리를 갱신해줘. | 신규 기능 + 기존 유지 |
| 11 | 대리결재 기능을 추가한다. (대규모 조건 변경) | 문맥 유지 한계 테스트 |
| 12 | 지금까지 등록된 전체 요구사항 목록을 정리해줘. | 누적 문맥 전체 유지 |
| 13 | 현재 승인 가능한 사용자는 누구인가? | 문맥 유지 + 환각 경계 |

### 3.2 환각 검증 질문 (5개)

| 번호 | 질문 | 정답 |
|------|------|------|
| H-1 | 이메일 자동 통보 기능이 존재하는가? | 존재하지 않음 |
| H-2 | 분양 외에 기탁신청하는 방법은 무엇인가? | 정의되어 있지 않음 |
| H-3 | 일반 사용자가 승인할 수 있는가? | 불가 (과장 이상만 가능) |
| H-4 | 첨부파일 등록 기능이 존재하는가? | 존재함 |
| H-5 | 반려 시 반려 사유 입력은 필수인가? | 필수 |

## 5. 모델별 결과 요약

### 5.1 ChatGPT (GPT-5.3 Auto)

**강점:**
- 예외 시나리오가 가장 풍부함 (ES-12개) — 동시성 문제, 파일 무결성, API 우회 공격까지 자발적으로 도출
- 테스트 케이스 구조가 명확 (입력/절차/기대결과 형식 일관성 높음)
- 환각 검증 답변이 시각적으로 명확 (✅/❌ 기호 활용)

**약점:**
- V1에서 요구사항 요약 섹션 없음
- 무료 버전으로 고급 추론 모델 자동 전환 제한 가능성

### 5.2 Claude (Sonnet 4.6)

**강점:**
- 페르소나 일관성이 가장 높음 — 이원규 역할을 13턴 내내 일관되게 유지
- 중복/모호한 요청에 대해 '이미 작성됨' 안내로 불필요한 반복 작업 방지
- 미확정 항목을 U-01~U-15로 번호화하여 추적 가능성과 재현 가능성이 가장 높음
- 요구사항 요약 섹션 포함으로 누락 사항 식별 강화

**약점:**
- V1에서 Few-shot 없이 형식 일관성이 다소 낮음

### 5.3 Gemini (3.5 Flash)

**강점:**
- V1에서 Given/When/Then 형식을 자발적으로 적용 — 수용 기준이 가장 상세
- 환각 검증 질문에서 단순 거절이 아닌 추가 요구사항 안내 제공

**약점:**
- V2에서 H-3 질문(일반 사용자 승인 여부)에 대해 Few-shot 출력 형식에 맞추려다 맥락을 잃어버리는 현상 발생
- 무료 버전으로 컨텍스트 길이 및 응답 속도 제한 가능성

## 6. 최종 선정 결론

### ✅ 최종 선정 모델: Claude Sonnet 4.6

**선정 근거 1: 페르소나 일관성과 재현 가능성이 가장 우수**

이원규 페르소나를 13턴 내내 일관되게 유지하며, 미확정 항목을 U-01~U-15로 번호화하여 체계적으로 관리하였다. 동일한 프롬프트를 재사용해도 동일한 품질의 산출물이 재현 가능하다는 점에서 업무 자동화 도구로서의 신뢰성이 가장 높다.

**선정 근거 2: 안전장치가 가장 효과적으로 작동**

환각 검증 H-1~H-5 모두 Pass를 기록하였으며, 특히 중복 요청 방지와 모호한 입력 처리에서 다른 모델 대비 명확한 우위를 보였다. 요구사항에 없는 기능을 질문받았을 때 단순 거절이 아닌 '현재 요구사항에 정의되어 있지 않습니다'라는 정형화된 응답으로 일관성을 유지하였다.

**선정 근거 3: 출력 품질의 완성도가 가장 높음**

V2 기준 5개 출력 섹션(요구사항 요약 / 추가 확인 필요 사항 / 사용자 스토리 / 수용 기준 / 테스트 케이스) 모두 완전하게 출력되었으며, ChatGPT와 동점(25/25)이나 페르소나 일관성과 미확정 항목 추적 체계에서 Claude가 우위를 보여 최종 선정하였다.
